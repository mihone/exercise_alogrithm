1 有没有遇到过jvm调试。
如果是要调参数的话，大多数情况合理配置是不需要调整的。
如果出现一些场景需要调优，可以从三个方面来排查具体问题
 第一，观察cpu指标和线程情况，分析热点代码或者有死锁的代码（jstack）
 第二，观察内存指标，看堆内存增长情况（jmap） 和mat
 第三看gc指标，比如ygc和fgc的耗时，次数。
通过上面三个方向确定具体代码问题或者参数配置问题。（比如bug，内存配置过小，堆内存过小）。
再比如如果元空间不够，可能就是反射太频繁。
2 有没用过分词器
主要采用默认的英文分词器+通过match phrase+设置相似距离来完成对中文的近似检索。
3 es集群是你自己搭的么？还是平台搭的
平台提供的能力是，帮忙往指定集群添加机器。
具体机器数量，配置，是我会根据数据量和每个文档大小预估总量，然后考虑到es多副本以及段合并占用内存，确定需要的大致分片数
然后确定分片数，从而确定的。
同时考虑到集群稳定性，会在不同中心申请，保证可用性和分区容错性。
