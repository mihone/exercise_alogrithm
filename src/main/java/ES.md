### 1 用过es的分词么
es本身提供了多种分词器
常见的有 1，默认分词器，按词切分，小写处理。2简单分词，按非字母切分，小写处理 3 按空白字符切分，保留原格式
对于中文，es需要额外安装中文分词器，常用的有ik_smart和ik_max_word,对应尽量少切分，和最大力度切分
### 2 如何配置自定义分词器
通过索引settings配置。
因此是索引级别的配置，不同索引想要用同一个分词器，需要分别配置。
### 3 分词器如何应用
定义字段类型的时候，可以追加指定analyzer，也可以在分词和搜索时使用不同的分词器
```json
{
  "mappings": {
    "properties": {
      "title": {
        "type": "text",
        "analyzer": "ik_max_word", // 为title字段指定IK分词器
        "search_analyzer": "ik_smart" // 搜索时使用不同的分词器
      },
      "description": {
        "type": "text",
        "analyzer": "my_custom_analyzer" // 为description字段指定自定义分词器
      }
    }
  }
}
```
### 4 不指定分词器，默认使用？
标准分词器，按照unicode进行分词，对中文的分词效果很差。
### 5 你怎么用分词
使用场景主要是对名称的搜索，使用ik_max_word，对中文分词效果更好，然后配置同义词词典。
并支持对词典的热更新，来保证搜索准确性。
另外就是索引和搜索采用不同的分词器，索引时用max_word搜索时用smart，避免搜索出与用户意图不相关的内容。
